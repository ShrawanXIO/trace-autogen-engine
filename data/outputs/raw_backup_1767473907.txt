### **Test Cases**

**Test Case ID: [TC_01]**
**Title:** Search by Valid Book Title
**Pre-conditions:**
- User is on the library search interface.
- The system accepts alphanumeric text input for Book Titles.
**Steps:**
1. Enter a valid Book Title (e.g., "The Great Gatsby") in the search bar.
2. Click the search button or press Enter.
**Expected Result:**
- The system displays all books matching the exact or partial title (case-insensitive).
- If no matches exist, display a "No results found" message.

**Test Case ID: [TC_03]**
**Title:** Search by Valid ISBN
**Pre-conditions:**
- User is on the library search interface.
- The system accepts numeric/hyphenated input for ISBNs (e.g., `978-0743273565` or `0743273565`).
**Steps:**
1. Enter a valid ISBN (e.g., `978-0743273565`) in the search bar.
2. Click the search button or press Enter.
**Expected Result:**
- The system displays the **exact book record** matching the ISBN.
- If invalid, display an error (e.g., "Invalid ISBN format").

---

**Test Case ID: [TC_04]**
**Title:** Combined Search (Title + Author + ISBN)
**Pre-conditions:**
- User is on the library search interface.
- The system supports multi-field input (Title, Author, ISBN).
**Steps:**
1. Enter a **Title** (e.g., "1984"), **Author** (e.g., "George Orwell"), and **ISBN** (e.g., `978-0451524935`) in the search bar.
2. Click the search button or press Enter.
**Expected Result:**
- The system displays **only the exact book** matching all three criteria.
- If no match exists, display "No results found."

---

**Test Case ID: [TC_05]**
**Title:** Partial Match Search (Title + Author)
**Pre-conditions:**
- User is on the library search interface.
- The system supports partial matches for Title and Author.
**Steps:**
1. Enter a **Title** (e.g., "Harry") and **Author** (e.g., "Rowling") in the search bar.
2. Click the search button or press Enter.
**Expected Result:**
- The system displays all books matching **both partial criteria** (e.g., titles containing "Harry" by authors containing "Rowling").
- If no matches exist, display "No results found."

---

**Test Case ID: [TC_06]**
**Title:** Pagination Boundary (Exactly 20 Items)
**Pre-conditions:**
- User performs a search returning **exactly 20 items**.
- The system enforces a 20-item-per-page limit.
**Steps:**
1. Execute a search that returns **20 books** (e.g., via a broad query like "fantasy").
2. Observe the displayed results.
**Expected Result:**
- All **20 items** are displayed on **Page 1**.
- No "Next Page" button is shown (since no overflow exists).

---

**Test Case ID: [TC_07]**
**Title:** Pagination Overflow (21+ Items)
**Pre-conditions:**
- User performs a search returning **21+ items**.
- The system enforces a 20-item-per-page limit.
**Steps:**
1. Execute a search that returns **21 books** (e.g., via a broad query like "mystery").
2. Observe the displayed results and pagination controls.
**Expected Result:**
- **First 20 items** are displayed on **Page 1**.
- The **21st item** is hidden and accessible only via a **"Next Page"** or **"Page 2"** button.
- Clicking "Next Page" displays the 21st item on **Page 2**.

---
**Notes:**
- All test cases assume the system handles **case-insensitive** searches unless specified otherwise.
- Edge cases (e.g., empty input, special characters in ISBNs) should be covered in additional test suites.
- Database validation (e.g., ISBN uniqueness) is implied but not explicitly tested here.
